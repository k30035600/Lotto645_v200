## 📎 가운데 패널 구성요소.docx

가운데 패널(Center Panel)

'선택공' 헤더, 필터, 번호 선택 그리드, 게임 생성 결과, 저장, 결과 확인 영역으로 구성되어 있습니다.

index-html의 구조와 app.js에 의해 동적으로 변경되는 구조를 종합하면 다음과 같습니다.

1. 헤더 및 필터 영역 (#numberGridHeaderContainer)

상단에 위치하며, "선택공" 타이틀과 필터 박스가 함께 배치되어 있습니다.

타이틀:"선택공"

필터 박스 (#filterBox):4가지 설정을 제공하여 생성될 번호의 속성을 제어합니다.

홀짝 필터:홀수와 짝수의 비율을 설정 (예: 4:2, 3:3 등).

연속 번호 필터:연속된 번호(예: 11, 12)의 포함 여부 설정.

핫/콜 필터:자주 나온 번호(Hot)와 안 나온 번호(Cold)의 비율 설정.

모드 선택수동 선택 모드로 전환 (게임 박스 비활성화).

자동/수동: 필터 조건에 따라 번호를 자동 생성하거나 반자동/수동 게임을 진행.

2. 번호 선택 그리드 (.number-grid-container)

UI 구성:1~45번까지의 로또 공 모양 버튼이 9열 5행 그리드로 배치됩니다.

정렬 기능:좌측 통계 패널의 정렬 기준(번호순, 당첨순, 출현순 등)에 따라 공의 배치 순서가 실시간으로 변경됩니다.

시각적 특징:당첨 횟수가 많은 상위 6개 번호에는 검은색 테두리가 표시되어 쉽게 식별할 수 있습니다.

기능:공을 클릭하여 번호를 선택하거나(수동 모드), 선호 번호로 등록/해제하는 등 상호작용합니다.

3. 게임 박스 (#gameBox)

게임 세트 컨테이너 (#gameSetsContainer): 총 5개의 게임(1게임~5게임)을 관리합니다.

각 게임 줄의 구성:

- 헤더:"1게임", "2게임" 등 라벨.

- 모드 버튼:각 게임별로 자동 / 반자동 / 수동 모드를 개별 전환할 수 있는 버튼.

- 합계 표시:생성된 6개 번호의 합계를 실시간으로 표시 (예: [ 135 ]).

- 번호 표시 영역:선택되거나 생성된 6개의 로또 공이 표시되는 영역.

- 체크박스:원하는 결과만 선택하여 저장하기 위한 체크박스.

4. 저장 박스 (#saveBox)

정보 입력:저장할 회차와 세트 번호를 표시하거나 입력받습니다.

저장 버튼:선택된(체크된) 게임 번호들을 아래 결과 박스에 저장 기록으로 남깁니다.

5. 결과 박스 (#resultBox)

기록 영역:저장 버튼을 통해 저장된 게임들이 리스트 형태로 누적되어 표시됩니다.

(초기 상태: "저장된 결과가 없습니다.")

1. 홀짝 (Odd/Even)

당첨 번호 6개 중 홀수와 짝수의 개수 비율을 검사합니다.

홀짝없음 (none):비율 제한 없음 (무조건 통과)

홀짝1:5 (1-5):홀수 1개, 짝수 5개 (코드에는 없으나 셀렉트 박스에 있다면 로직 확인 필요, 현재 코드상 매핑은 아래와 같음)

코드상 정의된 매핑 (ODD_EVEN_RATIO_MAP):

odd (홀수강세):홀수 4 : 짝수 2

even (짝수강세):홀수 2 : 짝수 4

balanced (균형):홀수 3 : 짝수 3

2. 연속 (Sequence)

정렬된 번호에서 연속된 숫자 쌍(예: 11-12)의 개수를 검사합니다.

연속없음 (none):연속된 쌍이 하나도 없어야 함 (0쌍)

연속1회 (1):연속된 쌍이 정확히 1개여야 함

연속2회 (2):연속된 쌍이 정확히 2개여야 함

3. 핫콜 (Hot/Cold)

과거 당첨 횟수 통계를 기준으로 상위 50%를 핫(Hot), 하위 50%를 **콜(Cold)**로 분류하여 비율을 검사합니다.

핫콜없음 (none):비율 제한 없음

핫콜1:5 (1-5):(UI 옵션값, 코드 매핑 확인 필요)

코드상 정의된 매핑 (HOT_COLD_RATIO_MAP):hot (핫 강세): 핫 4 : 콜 2

cold (콜 강세): 핫 2 : 콜 4

mixed (균형): 핫 3 : 콜 3

참고

index.html의 옵션 값(1-5, 2-4 등)과 app.js의 매핑 키(odd, even 등)가 일치하지 않는 부분이 보입니다. 현재 app.js 코드는 ODD_EVEN_RATIO_MAP 객체를 사용해 odd, even, balanced 키를 찾고 있는데, 실제 HTML select 값은 1-5, 2-4 등으로 되어 있어 매핑 수정이 필요할 수 있습니다.

요청하신 영상 **"How to Improve Your Chances of Winning the Lottery Using Python"**의 주요 내용을 요약해 드립니다. 이 영상은 파이썬(Python) 프로그래밍을 활용해 과거 로또(파워볼) 당첨 번호를 분석하고, 통계적으로 당첨 확률을 높이는 방법을 다루고 있습니다.

1. 영상 개요 및 목적

- 로또 당첨 확률(약 2억 9,200만 분의 1)은 매우 희박하지만, 데이터 분석과 패턴 인식을 통해 무작위 선택보다 조금 더 유리한 조합을 찾을 수 있다고 주장합니다.

- 모든 번호가 홀수이거나 짝수인 경우, 혹은 1, 2, 3, 4, 5처럼 연속된 번호가 나오는 극단적인 패턴은 당첨 확률이 매우 낮으므로 배제하는 전략을 사용합니다.

2. 주요 분석 단계 및 방법

① 데이터 수집 (Web Scraping)

- Selenium과 BeautifulSoup 라이브러리를 사용하여 웹사이트에서 과거 파워볼 당첨 번호를 자동으로 수집(크롤링)합니다.

- 수집한 데이터는 Pandas DataFrame으로 정리하여 분석하기 쉬운 형태로 만듭니다.

② 빈도 분석 (Frequency Analysis)

- 최근 1년여간의 데이터를 분석한 결과, 특정 번호(예: 숫자 9)는 약 30회 출현한 반면, 어떤 번호(예: 숫자 49)는 8~9회밖에 나오지 않는 등 출현 빈도의 편차가 있음을 확인했습니다.

- 이러한 '자주 나오는 번호'와 '잘 안 나오는 번호'의 데이터를 전략에 활용합니다.

③ 패턴 및 규칙 설정 (Rules & Guardrails)

- 합계 구간: 당첨 번호 5개의 합계가 대부분 100~250 사이에 분포하며, 특히 140~180 구간에 집중된다는 점을 발견했습니다. 극단적으로 낮거나 높은 합계는 제외합니다.

- 홀짝/고저 비율: 홀수와 짝수, 낮은 수와 높은 수가 적절히 섞인 균형 잡힌 조합이 당첨될 확률이 높습니다.

3. 번호 생성 알고리즘

- 단순한 무작위 생성이 아니라, 위에서 설정한 통계적 규칙(합계 범위, 홀짝 비율 등)을 통과하는 번호 조합만 남기는 방식으로 예상 번호를 생성합니다.

- 영상에서는 이 알고리즘을 통해 20개의 추천 게임 조합을 생성하는 시연을 보여줍니다.

4. 결론

- 이러한 분석이 당첨을 보장하지는 않지만, "승산 없는 패턴(Dead Patterns)"을 피하고 통계적으로 유의미한 조합에 투자함으로써 확률을 조금이라도 높이는 데 의의가 있습니다.

- 개발자는 이 과정이 데이터를 다루는 재미있는 학습 과정임을 강조하며 마무리합니다.

이 영상의 접근 방식(과거 데이터 크롤링 -> 빈도/패턴 분석 -> 조건부 번호 생성)은 앞서 제가 1203회 예상 번호를 분석해 드린 논리와 매우 유사합니다.
